Video.js 02 选项参考

发表于 2023-06-29 阅读次数：

Video.js 选项参数包括:

  • <video> 元素标准选项
  • Video.js 特定选项
  • 组件选项和技术选项

<video> 元素标准选项

autoplay

类型: boolean|string

自动播放,有效值包括:

值 类型 说明
flase boolean 与不设置此值含义相同,不自动播放
true boolean 与此元素的原生值含义相同,自动播放
muted boolean 使视频元素静音，然后调用play(),这使自动播放更可能奏效。
play boolean 调用play(),与原生autoplay含义相同
any boolean 先尝试调用play(),如果失败,则静音后再次调用play()

选项设置示例

controlBar.remainingTimeDisplay.displayNegative

默认剩余时间显示为负时间。controlBar.remainingTimeDisplay.displayNegative 设置为 false ,不显示负号。

controls

类型: boolean

是否显示交互控件。如果没有控件，则只能使用autoplay属性或通过播放器 API 开始播放视频。

height

类型: string|number

设置视频播放器的显示高度（以像素为单位）。

loop

类型：boolean

使视频在结束时立即重新开始。

muted

类型：boolean

静音。

poster

类型：string

在视频开始播放之前显示的图像的 URL。这通常是视频的帧或自定义标题屏幕。一旦用户点击“播放”，图像就会消失。

preload

类型：string

向浏览器建议是否应在加载 <video> 元素后立即开始下载视频数据。支持的值包括：

  • ‘auto’
    立即开始加载视频（如果浏览器支持）。某些移动设备不会预加载视频以保护用户的带宽/数据使用。这就是为什么该值被称为“自动”而不是像’true’.这往往是最常见和推荐的值，因为它允许浏览器选择最佳行为。

  • ‘metadata’
    仅加载视频的元数据，其中包括视频的持续时间和尺寸等信息。有时，元数据将通过下载几帧视频来加载。

  • ‘none’
    不预加载任何数据。浏览器将等到用户点击“播放”开始下载。

width

类型：string|number

设置视频播放器的显示宽度（以像素为单位）。

Video.js 特定选项

除非另有指定，否则每个选项的默认值都是 undefined。

aspectRatio

类型：string

将播放器置于动态模式，并在计算播放器的动态大小时使用该值。该值应表示一个比率 - 两个用冒号分隔的数字（例如 “16:9” 或 “4:3” ）。

或者，可以将样式类 vjs-16-9,vjs-9-16,vjs-4-3 或 vjs-1-1 添加到播放器中。

audioOnlyMode

类型：boolean 默认：false

如果设置为 true，它将异步隐藏除控制栏之外的所有播放器组件，以及仅视频所需的任何特定控件。此选项可以设置为true或false在运行audioOnlyMode([true|false])时调用。当用作二传手时，它返回一个 Promise。当用作 getter 时，它返回一个布尔值。

audioPosterMode

类型：boolean 默认：false

如果设置为 true，则通过隐藏视频元素并持久显示海报图像来启用海报查看器体验。此选项可以设置为true或false在运行audioPosterMode([true|false])时调用。

autoSetup

类型：boolean

阻止播放器对具有data-setup属性的媒体元素运行自动设置。

注意：这必须在加载 videojs 源的同一刻度中进行全局设置videojs.options.autoSetup = false才能生效。

breakpoints

类型：Object

与响应选项一起使用时，设置断点，这些断点将配置如何在播放器上切换样式类名，以根据播放器的尺寸调整 UI。

children

类型：Array|Object

此选项继承自组件基类。

disablePictureInPicture

类型：boolean

如果 true，则禁用将视频元素切换为画中画。默认值为 false。

这对 Firefox 没有影响，Firefox 实现了专有的画中画模式，该模式没有实现标准的画中画 API。

这不会禁用允许播放器元素放入 PiP 窗口的文档画中画模式。

enableDocumentPictureInPicture

类型：boolean

如果 true，则文档画中画 API 将用于画中画（如果可用）。默认为 false，但当功能已建立时默认为true。

目前，这在 Chrome 111 +中作为原始试用版可用。

这是一种与以前可用的画中画模式不同的模式，其中整个播放器元素都是窗口化的，而不仅仅是视频本身。由于在某些情况下，最好在播放器上允许画中画，而不允许仅在视频上允许画中画（例如广告、叠加），因此disablePictureInPicture选项仅禁用视频上的旧式画中画模式。

experimentalSvgIcons

类型：boolean

如果设置为 true，则在整个播放器中使用的 videojs/font 图标将被存储在 Video.js 中的 SVG 替换。默认为 false，但当功能已建立时默认为true。

这些 SVG 是从 Font Awesome 和 Google 的 Material UI 下载的。

您可以通过将 sandbox/svg-icons.html.example 重命名为 sandbox/svg-icons.html、使用 npm run build构建 Video.js 并在您选择的浏览器中打开sandbox/svg-icons.html来查看所有可用的图标。

自定义播放器时，应使用 setIcon 方法将图标添加到播放器内部Component。

fluid

类型：boolean

当 true 时，Video.js 播放器将具有流体大小。换句话说，它将以视频的固有纵横比或指定的纵横比缩放以适应其容器。
此外，如果<video>元素具有 “vjs-fluid”，则此选项会自动设置为 true。

fullscreen

类型：Object 默认：{options: {navigationUI: 'hide'}

fullscreen.options 可以设置为传入特定的全屏选项。在某些时候，它将通过element 和 handler 进行功能增强。

有关更多详细信息，请参阅全屏 API 规范。

id

类型：string

如果提供，并且该元素还没有 id，则此值用作播放器元素的 id。

inactivityTimeout

类型：number

Video.js 表示用户通过 "vjs-user-active" 和 "vjs-user-inactive" 样式类和 "useractive" 事件与播放器进行交互。

inactivityTimeout 确定在声明用户处于非活动状态之前需要多少毫秒的非活动时间。值 0 表示没有 inactivityTimeout 并且用户永远不会被视为非活动状态。

language

类型：string，默认：浏览器默认或 'en'

与播放器中的可用语言之一匹配的语言代码。这将设置玩家的初始语言，但始终可以更改。

如需详细了解，请参阅 video.js 语言。

languages

类型：Object

自定义播放器中可用的语言。此对象的键将是语言代码，值将是具有英语键和翻译值的对象。

如需详细了解，请参阅 video.js 语言。

注意：通常，此选项不是必需的，最好将您的自定义语言传递给 videojs.addLanguage()，以便它们在所有播放器中可用！

liveui

类型：boolean 默认：false

允许玩家使用新的实时 UI，其中包括：

  • 用于在实时窗口中查找的进度条
  • 可以单击该按钮以查找活动边缘，并带有一个圆圈，指示您是否在活动边缘。

liveTracker.trackingThreshold

类型：number 默认：20

播放器的实时跟踪器组件的一个选项，用于控制何时显示 liveui。默认情况下，如果流在搜索栏上的时间少于 20 秒，那么即使设置了 liveui 选项，我们也不会显示新的 liveui。

liveTracker.liveTolerance

类型：number 默认：15

播放器的 liveTracker 组件的一个选项，用于控制距可搜索终点的距离，应被视为实时播放。默认情况下，距离实时可寻道边超过 15 秒的任何内容都被视为实时，其他所有内容都被视为实时。任何向后搜索的用户交互都将忽略此值，正如用户所期望的那样。

nativeControlsForTouch

类型：boolean

显式设置关联技术选项的默认值。

normalizeAutoplay

类型：boolean

指定是否应将 set autoplay: true和 <video autoplay> 视为与 autoplay: 'play'相同，即应从autoplay视频元素中删除（或不添加到）该属性，并由 Video.js 手动启动 play() 而不是浏览器。

notSupportedMessage

类型：string

允许覆盖当 video.js 无法播放媒体源时显示的默认消息。

noUITitleAttributes

类型：boolean 默认：false

控制 UI 元素是否具有title属性。title属性显示在鼠标悬停时，这有助于提高可用性，但对辅助功能有缺点。设置noUITitleAttributes为true防止将title属性添加到 UI 元素，从而允许插件或外部框架将更易于访问的工具提示添加到控件中。

playbackRates

类型：Array

严格大于 0 的数字数组，其中 1 表示常规速度 （100%），0.5 表示半速（50%），2 表示双速（200%）等。 如果指定，video.js 将显示一个控件（vjs-playback-ratey 样式类） 允许用户从一系列选择中选择播放速度。 选项按指定顺序从下到上显示。

preferFullWindow

类型：boolean， 默认值：false

将此项设置为 true 将更改不支持 HTML5 全屏 API 但支持视频元素全屏的设备（即 iPhone）上的全屏行为。播放器不会被全屏显示，而是拉伸播放器以填满浏览器窗口。

responsive

类型：boolean，默认值：false

将此选项设置为 true 将导致播放器根据响应断点自定义自身（请参阅：断点选项）。

当此选项为 false（默认值） 时，将忽略响应断点。

请注意，这是关于播放器内控件的响应能力，而不是播放器本身的响应大小。为此，请参阅流体。

restoreEl

类型: boolean|Element，默认值：false

如果设置为 true，将在启动播放器之前创建占位符元素的副本。如果播放器被释放，副本将放回玩家位置的 DOM 中。

如果设置为 HTML 元素，则该元素将替换已释放的播放器。

skipButtons

类型：Object

skipButtons.forward

类型：number

如果指定，Video.js 将显示一个控件，允许 用户在视频中向前跳转指定的秒数。

以下值有效：5 | 10 | 30

skipButtons.backward

类型：number

如果指定，Video.js 将显示一个控件，允许 用户在视频中按指定的秒数跳回。

以下值有效：5 | 10 | 30

sources

类型：Array

一个对象数组，这些对象反映原生 <video> 元素具有一系列<source>子元素的功能。这应该是具有 src 与 type 属性的对象数组。例如：

suppressNotSupportedError

类型：boolean

如果设置为 true，则不会立即触发“无兼容源”错误，而是在第一次用户交互时发生。这对于 Google 的“移动友好”测试工具很有用，该工具无法播放视频，但您可能不希望看到显示的错误。

techCanOverridePoster

类型：boolean

让技术人员可以覆盖播放器的海报 并整合到播放器的海报生命周期中。 当使用多个播放器并且每个技术人员都必须设置自己的海报时，这可能很有用 每当播放新源时。

techOrder

类型：Array，默认值：['html5']

定义首选 video.js 技术的顺序。默认情况下，这意味着 Html5 技术是首选。其他注册技术人员将按照注册顺序添加到此技术之后。

userActions

类型：Object

userActions.click

类型：boolean|function

控制点击播放器/技术的操作方式。如果设置为 false，则单击将被禁用，并且不再导致播放器在暂停和播放之间切换。如果使用 controls: false 禁用控件，则不会调用处理程序函数。

userActions.doubleClick

类型：boolean|function

控制双击玩家/技术的操作方式。如果设置为 false，则禁用双击。如果未定义或设置为 true，则启用双击并切换全屏模式。要覆盖默认的双击处理，请设置 userActions.doubleClick 为接受事件的 dblclick 函数：

userActions.hotkeys

类型：boolean|function|object

控制播放器范围的热键的操作方式。如果设置为 false 或 undefined，则禁用热键。如果设置为 true 或者对象（以允许在下面定义 fullscreenKey 等.），则按如下所述启用热键。若要覆盖默认热键处理，请设置 userActions.hotkeys 为接受事件的函数。如果使用 keydowncontrols: false 禁用控件，则不会调用处理程序函数。

默认热键处理为：

按键 动作 启用
f 切换全屏 仅当控制栏中存在全屏按钮时才启用
m 切换静音 始终启用，即使不存在控制栏
k 切换播放/暂停 始终启用，即使不存在控制栏
Space 切换播放/暂停 始终启用，即使不存在控制栏

热键首先需要播放器获取焦点。请注意，如果 Space 按钮和菜单等控件具有键盘焦点，则该键将激活该控件。无论播放器中的哪个控件具有焦点，其他热键都有效。

userActions.hotkeys.fullscreenKey

类型：function

覆盖全屏键定义。如果设置了此选项，则函数接收keydown事件;如果函数返回 true，则执行全屏切换操作。

userActions.hotkeys.muteKey

类型：function

覆盖静音键定义。如果设置了此选项，则函数接收 keydown 事件;如果函数返回 true，则执行静音切换操作。

userActions.hotkeys.playPauseKey

类型：function

覆盖播放/暂停键定义。如果设置了此选项，则函数接收 keydown 事件;如果函数返回 true，则执行播放/暂停切换操作。

vtt.js

类型：string

允许将默认 URL 覆盖到 vtt.js，该 URL 可以异步加载以填充对 WebVTT 的填充代码支持 。

此选项将用于视频的“novtt”构建.js（即 video.novtt.js）。否则，vtt.js 与 Video.js 捆绑在一起。

组件选项

Video.js 播放器是一个组件。与所有组件一样，您可以定义它包含哪些子组件、它们出现的顺序以及传递给它们的选项。

这是一个快速参考;因此，有关 Video.js 中组件的更多详细信息，请查看组件指南。

类型：Array|Object

技术选项

${techName}

类型：Object

Video.js 播放技术（”techs”）可以作为传递给 videojs 函数的选项的一部分提供自定义选项。它们应该在技术名称的小写变体下传递（例如 “html5”）。

html5

nativeControlsForTouch

类型：boolean

仅受 Html5 技术支持，此选项可以设置为 true 强制触摸设备的本机控件。

nativeAudioTracks

类型：boolean

可以设置为 false 以禁用本机音轨支持。最常与 videojs-contrib-hls 一起使用。

nativeTextTracks

类型：boolean

可以设置为 false 强制模拟文本轨道而不是本机支持。nativeCaptions 选项也存在，但只是 nativeTextTracks 的别名。

nativeVideoTracks

类型：boolean

可以设置为 false 以禁用本机视频轨道支持。最常与 videojs-contrib-hls 一起使用。

preloadTextTracks

类型：boolean

可以设置为 false 延迟加载非活动文本轨道，直到使用。这可能会导致切换字幕时出现短暂的延迟，在此期间可能会缺少字幕。

默认行为是预加载所有文本轨道。
